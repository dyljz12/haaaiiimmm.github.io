Mysql启动报错

post @ 2016-05-11

错误信息: The server quit without updating PID file

尝试解决

google了一下解决办法太多。随便找了几个方法尝试无果。

解决办法包括：

检查是否有僵尸进程，kill掉再重启即可。

ps -a | grep mysql确实发现有mysql进程，杀掉后再mysql.server start，无法启动，报了同样的错，WTF。

再检查进程，那个该死的进程还在，只是换了个进程号，再检查，依旧。

分析了问题所在：mysql守护进程其实一直在重启mysql，只不过因为不明原因启动失败，我杀掉再重启不过是干了同样事，当然没解决问题。

在/etc/my.cnf 中加入
‘’’
datadir = /usr/local/mysql/data
log-error = /usr/local/mysql/data/error.log
pid-file = /usr/local/mysql/data/mysql.pid
user = mysql
tmpdir = /tmp
‘’’

半信半疑的加上了这几个配置，没解决。

用户没有xxx. pid这个文件的的写权限

看上去就不合理，难道只有root 能玩数据库吗。

但还是试了下 chomd 给自己全部权限。

无果。。。。

删掉所有err 文件

看起来很搞笑，居然还有人说这办法可行。

好吧我也试试。

到/usr/local/mysql/data中rm了所有err文件。

显然没用。而且因为守护进程的重启又生成了err文件。
